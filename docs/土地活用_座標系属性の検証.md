# 土地活用推進調査：座標系属性のサンプル検証

## 目的

マージ時に「どの属性が座標系（平面直角 1～19 系）か」を特定する。  
測地系(1/2) や他の数値コードと区別するため、サンプルで属性一覧を確認する。

## 手順

### 1. 土地活用の GPKG を用意する

まだない場合は、先に CSV → GeoPackage 変換を実行する。

```bash
cd /path/to/gdal-full
source env.sh
bash scripts/csv_to_geopackage.sh
```

これで `データ_geopackage_converted/土地活用推進調査/*.gpkg` ができる。

### 2. 検証スクリプトを実行する

```bash
source env.sh
python3 scripts/verify_tochi_attributes.py
```

ファイルを指定する場合:

```bash
python3 scripts/verify_tochi_attributes.py "inputfile/.../データ_geopackage_converted/土地活用推進調査/TH_01103.gpkg"
```

ogrinfo の生出力を残す場合:

```bash
python3 scripts/verify_tochi_attributes.py -o tochi_ogrinfo_sample.txt
```

### 3. 出力の見方

- **スキーマ**: Field 番号と属性名（GPKG によっては出ない形式もあります）
- **先頭レコードの属性一覧**: `idx | 属性名 | 値` の表
- 値が **1～19** の行には `[座標系候補]` と表示される

### 4. 座標系列の判定

- **座標系**: 平面直角の「系」で、1～19 の整数。ファイル（地域）ごとに 2, 9, 12 などが混在する。
- **測地系**: 1 または 2 が多く、多くのファイルで 2（JGD2011）。
- 複数ファイルで実行し、「地域が変わると値が変わる列」が座標系の可能性が高い。  
  例: TH_01103（青森）→ 12、九州のファイル → 2 や 7 など。

検証で座標系と判断した列の **idx（0 始まり）** が分かったら、`merge_tochi_geopackage.py` でそのインデックスを優先して使うよう修正できる。
