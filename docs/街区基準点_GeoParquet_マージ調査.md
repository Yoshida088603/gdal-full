# 街区基準点等データ GeoParquet マージ・CRS 統一 調査

## 目的

- **出力先**: `inputfile/.../データ_geoparquet_marged`（ユーザ指定の格納先）
- **入力**: `データ_geoparquet_converted/街区基準点等データ/*.parquet`（3,594 ファイル）
- **作業**: 全ファイルを 1 本の GeoParquet にマージし、**CRS を MapLibre 用（WGS84）に統一**する。

---

## 1. 変換元 CSV の構造（街区基準点等データ）

サンプル（H_15100.csv, H_13123.csv, H_27227.csv）より:

| 列番号(0始まり) | ヘッダ名（例） | 内容・備考 |
|-----------------|----------------|------------|
| 0 | 補正情報 | 補正/空等 |
| 1 | 補正後X座標 | 補正後 X |
| 2 | 補正後Y座標 | 補正後 Y |
| 3 | 補正後標高 | 補正後標高 |
| 4 | 廃点情報 | 廃点/空 |
| 5 | 市区町コード | 例: 15100, 13123, 27227 |
| 6 | 市区町名 | 例: 新潟市, 船橋市, 浦安市 |
| 7 | 旧市区町コード | |
| 8 | 旧市区町名 | |
| 9 | 街区点設置者種別 | 0 等 |
| 10 | 街区点・補助点名称 | 例: 5A001, 3A001, 4A003 |
| **11** | **座標系** | **1～19（平面直角 I～XIX 系）** |
| **12** | **測地系** | **2 = JGD2011** |
| **13** | **X座標** | 平面直角 X（メートル） |
| **14** | **Y座標** | 平面直角 Y（メートル） |
| 15 | 標高 | |
| 16 | 地盤高 | |
| 17 | 所在地 | 住所文字列 |
| 18 | 材質区分 | |
| 19 | 測量年月日 | YYYYMMDD |
| 20 | 測定方式 | |
| 21 | 基準点コード | 例: KG3152015A001 |
| 22 | 現況地目 | |

- **ジオメトリ**: 点（1 行 = 1 点）。座標は **X座標**, **Y座標** のみ使用（補正後は使わず）。
- **エンコーディング**: UTF-8（H_15100 等）と CP932（H_01101 等）が混在。街区は `csv_to_geoparquet.sh` で iconv を通さずそのまま ogr2ogr するため、Parquet の属性名が環境により文字化けすることがある。マージスクリプトは「最初の OGRFeature の 12 番目属性の値」で座標系を取得するフォールバックで対応（値が空の行も 1 行として数えるため正規表現は `= (.*)` を使用）。

## 2. 第1段階変換後の GeoParquet（現状）

- **作成方法**: `csv_to_geoparquet.sh` で `ogr2ogr -f Parquet -oo X_POSSIBLE_NAMES=X座標 -oo Y_POSSIBLE_NAMES=Y座標` により 1 CSV → 1 Parquet。
- **CRS**: **未指定**（`-s_srs` / `-t_srs` なし、.prj なし）のため、**座標系は Parquet に記録されていない**可能性が高い。属性として **座標系**（列 11）・**測地系**（列 12）はそのまま格納されている。
- **レイヤ名**: GDAL Parquet ドライバのデフォルト（例: ファイル名ベースのレイヤ 1 つ）。
- **属性**: 上記 CSV の列がそのまま属性として入る（X座標, Y座標, 座標系, 測地系 等）。
- **ジオメトリ**: Point（WKB）。座標値は **平面直角座標のまま**（メートル）。

## 3. CRS の対応関係

### 3.1 座標系列（1～19）と EPSG

| 座標系（列値） | 平面直角 | EPSG（JGD2011） |
|----------------|----------|------------------|
| 1 | I 系 | EPSG:6669 |
| 2 | II 系 | EPSG:6670 |
| … | … | … |
| 6 | VI 系 | EPSG:6674 |
| 7 | VII 系 | EPSG:6675 |
| 8 | VIII 系 | EPSG:6676 |
| 9 | IX 系 | EPSG:6677 |
| … | … | … |
| 19 | XIX 系 | EPSG:6687 |

- **測地系 2** = JGD2011。上記 EPSG はすべて JGD2011 ベース。
- **式**: 座標系 \(z\)（1～19）→ `EPSG:6668 + z`（6669～6687）。

### 3.2 MapLibre で使う CRS

- **WGS84 経緯度**: **EPSG:4326**。  
  MapLibre GL は GeoJSON やベクタータイルの座標を **経緯度（EPSG:4326）** として解釈するため、マージ後の GeoParquet は **EPSG:4326** に統一する。

---

## 4. ファイルごとの座標系

- 街区基準点の CSV は **1 ファイルが 1 市区町村コード** に対応し、そのファイル内の **座標系は同一**（全レコードで「座標系」列が同じ値）。
- サンプル:
  - H_15100.csv → 座標系 **8**（新潟付近、VIII 系）
  - H_13123.csv → 座標系 **9**（船橋付近、IX 系）
  - H_27227.csv → 座標系 **6**（浦安付近、VI 系）
- したがって **ファイル単位で `-s_srs EPSG:6669～6687` を決め、`-t_srs EPSG:4326` で変換**すればよい。

---

## 5. マージ手順（方針）

1. **出力先**: `データ_geoparquet_marged` 直下に、例: `街区基準点等_merged.parquet` を出力。
2. **CRS 変換**:  
   各 Parquet について  
   - そのファイルの **座標系** を 1 レコード目（または全件で共通のため任意の 1 件）の属性から取得。  
   - 対応する EPSG を `6668 + 座標系` で算出（1～19 → 6669～6687）。  
   - `ogr2ogr -s_srs EPSG:<上記> -t_srs EPSG:4326` で WGS84 に変換し、一時 GeoPackage または GeoJSON に書き出し、  
   - それらを 1 本の Parquet に **マージ**（`ogr2ogr` で複数ソースを順に append、または GDAL の VRT でマージしてから Parquet 出力）。
3. **スキーマ**: 属性は現状のまま**すべて保持**する（`-select` で列を削らない）。**元の X座標・Y座標・標高（および補正後X座標・補正後Y座標・補正後標高）は属性として完全に残る**。ジオメトリのみ WGS84 に変換され、経緯度が geometry 列に入る。
4. **検証**: マージ後、`ogrinfo -al -so 街区基準点等_merged.parquet` で  
   - Layer の CRS が **EPSG:4326**  
   - Feature Count が元の 3,594 ファイルの合計件数と一致することを確認。

---

## 6. 実装時の注意

- **座標系が欠損しているレコード**: そのファイルの他レコードの座標系で補完するか、欠損時はスキップするか方針を決める。
- **測地系**: 現状は 2（JGD2011）のみ想定。将来 1（旧測地系）が混在する場合は、測地系に応じた EPSG テーブルが必要。
- **出力ディレクトリ名**: `データ_geoparquet_marged` を使用。

## 7. まとめ

| 項目 | 内容 |
|------|------|
| 入力 | データ_geoparquet_converted/街区基準点等データ/*.parquet（3,594 ファイル） |
| 出力 | データ_geoparquet_marged/街区基準点等_merged.parquet（1 ファイル） |
| 入力 CRS | ファイルごとに属性「座標系」(1～19) → EPSG:6669～6687（JGD2011 平面直角） |
| 出力 CRS | **EPSG:4326（WGS84）**（MapLibre 用） |
| ジオメトリ | Point（経緯度に変換） |
| 属性 | 既存を維持（座標系・X座標・Y座標 等も保持可） |

**実装済み**: [scripts/merge_gaiku_geoparquet.py](../scripts/merge_gaiku_geoparquet.py) で CRS 統一＋マージを実行する。  
**実行例**: gdal-full をカレントに `source env.sh` したうえで `python3 scripts/merge_gaiku_geoparquet.py`。既存の `街区基準点等_merged.parquet` は上書きされる。座標系が取得できないファイルは警告してスキップする。全件処理後に `ogrinfo -al -so` で CRS（EPSG:4326）と Feature Count を表示する。

**座標系の取得**: スクリプトは (1) まず属性名「座標系」で `ogrinfo -al` 出力から値を取得する。(2) マッチしない場合はスキーマの「Field 11」の列名を取得し、その名前で値を探す（Parquet で列名が別名・英語化されている場合のフォールバック）。最初にスキップされたファイルがある場合、そのファイルの ogrinfo 先頭 80 行を stderr に出力するので、実際のフィールド名を確認できる。
