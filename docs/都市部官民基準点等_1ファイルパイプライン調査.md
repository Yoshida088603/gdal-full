# 都市部官民基準点等データ：1ファイル CSV → GeoPackage → PMTiles パイプライン調査

## 対象ファイル

- **TKS_04206.csv**（白石市、都市部官民基準点等データ）
- パス: `データ_origin/都市部官民基準点等データ/TKS_04206.csv`

## CSV の中身

- **文字コード**: CP932（Shift_JIS）
- **ヘッダ**: 1行目あり。主要列は以下のとおり。

| 列番(1-based) | ヘッダ名   | 備考 |
|---------------|------------|------|
| 12            | 座標系     | 平面直角座標系の系番号（1–19）。本ファイルは **10** → EPSG:6678 |
| 14            | X座標      | 平面直角座標（メートル） |
| 15            | Y座標      | 平面直角座標（メートル） |

- 街区基準点等データと同様に「座標系」「X座標」「Y座標」列を持つため、既存の `csv_to_geoparquet_tochi.py` で街区と同じ扱い（UTF-8 化・座標系取得・X/Y ヘッダ入れ替え）が可能。

## 1ファイルパイプラインの実行結果

既存の **single_csv_to_pmtiles.sh** をそのまま使用して実行した。

```bash
cd gdal-full
source env.sh
bash scripts/single_csv_to_pmtiles.sh "inputfile/.../データ_origin/都市部官民基準点等データ/TKS_04206.csv"
```

- **結果**: 成功。
- 座標系 10 → EPSG:6678 で GeoPackage 作成後、EPSG:3857 に変換して PMTiles まで出力。
- 出力:
  - `データ_geopackage_marged/TKS_04206.gpkg`（レイヤ名: `merged`、Geometry: Point、5 件）
  - `データ_geopackage_marged/TKS_04206.pmtiles`

## 結論

- **都市部官民基準点等の 1 ファイル CSV でも、現状の「1ファイル CSV → GeoPackage → PMTiles」パイプライン（single_csv_to_pmtiles.sh）がそのまま利用可能。**
- 街区基準点等と同様に「座標系」「X座標」「Y座標」の列構成のため、追加対応は不要。
